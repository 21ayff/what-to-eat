# 🍽️ 今天吃什么 - 需求文档

## 1. 功能需求

### 1.1 用户系统
- **用户注册**：支持邮箱/手机号注册
- **用户登录**：支持邮箱+密码、手机号+验证码登录
- **密码管理**：找回密码、修改密码
- **个人资料**：头像、昵称、性别、生日、饮食偏好设置
- **账号安全**：JWT Token 认证、Token 刷新机制

### 1.2 美食推荐系统
- **智能推荐算法**：基于用户历史偏好、时段、天气、心情等多维度推荐
- **推荐历史**：记录用户的推荐历史，支持查看和重新推荐
- **收藏功能**：用户可以收藏喜欢的美食
- **评分反馈**：用户对推荐结果进行评分，优化后续推荐

### 1.3 美食数据管理
- **美食库**：216+ 道美食数据，支持分类、标签、营养成分
- **美食搜索**：按名称、菜系、食材、口味搜索
- **美食详情**：图片、做法、营养信息、适宜人群
- **管理员后台**：添加、编辑、删除美食数据

### 1.4 趣味游戏
- **美食转盘**：随机选择美食，支持自定义选项
- **骰子挑战**：掷骰子决定吃什么
- **快问快答**：通过回答问题快速缩小选择范围
- **游戏记录**：记录游戏结果，生成统计报告

### 1.5 个人中心
- **探索历程**：统计尝试过的美食数量、种类
- **成就系统**：完成特定任务获得徽章
- **饮食报告**：周/月饮食分析报告
- **偏好设置**：忌口、过敏源、饮食目标设置

### 1.6 社交功能（可选）
- **分享功能**：分享推荐结果到社交媒体
- **好友系统**：查看好友的饮食动态
- **群组功能**：聚餐时群组决策

## 2. 非功能需求

### 2.1 性能需求
- **响应时间**：API 响应时间 < 200ms（P95）
- **并发处理**：支持 1000+ 并发用户
- **页面加载**：首屏加载 < 2s
- **推荐算法**：推荐计算 < 500ms

### 2.2 可用性需求
- **系统可用性**：99.9% 可用性（年停机时间 < 8.76 小时）
- **故障恢复**：自动故障转移，数据备份
- **降级策略**：服务异常时提供基础功能

### 2.3 安全性需求
- **数据加密**：敏感数据加密存储（密码 bcrypt）
- **传输安全**：HTTPS 全站加密
- **访问控制**：基于角色的权限管理
- **防攻击**：防 SQL 注入、XSS、CSRF
- **限流保护**：API 请求限流，防暴力破解

### 2.4 可扩展性需求
- **水平扩展**：支持多实例部署
- **微服务就绪**：架构支持未来拆分为微服务
- **缓存策略**：Redis 缓存热点数据

### 2.5 兼容性需求
- **浏览器支持**：Chrome、Firefox、Safari、Edge 最新 2 个版本
- **移动端**：iOS Safari、Android Chrome
- **响应式**：适配手机、平板、桌面

## 3. 用户需求

### 3.1 目标用户画像

#### 用户类型 A：上班族（主要用户）
- **年龄**：22-35 岁
- **痛点**：工作忙碌，没时间想吃什么
- **需求**：快速决策、健康推荐、外卖友好
- **使用场景**：工作日午餐、加班晚餐

#### 用户类型 B：学生党
- **年龄**：18-25 岁
- **痛点**：预算有限，选择困难
- **需求**：便宜好吃、周边推荐、聚餐决策
- **使用场景**：食堂选择、宿舍聚餐

#### 用户类型 C：家庭主妇/主夫
- **年龄**：30-50 岁
- **痛点**：每天做饭不知道做什么
- **需求**：营养均衡、食材搭配、菜谱学习
- **使用场景**：日常三餐、家庭聚餐

#### 用户类型 D：美食爱好者
- **年龄**：25-40 岁
- **痛点**：想尝试新美食但不知道选什么
- **需求**：探索新菜系、记录美食体验、分享
- **使用场景**：周末探店、旅行美食

### 3.2 用户故事

1. **作为**上班族，**我希望**快速获得午餐推荐，**以便**节省时间
2. **作为**用户，**我希望**根据天气获得推荐，**以便**吃得舒适
3. **作为**用户，**我希望**记录我的饮食偏好，**以便**获得更精准的推荐
4. **作为**用户，**我希望**收藏喜欢的美食，**以便**下次快速找到
5. **作为**用户，**我希望**和朋友一起决定吃什么，**以便**解决聚餐选择困难
6. **作为**管理员，**我希望**管理美食数据，**以便**保持内容新鲜

## 5. 数据库需求

### 5.1 数据实体

#### 用户表 (users)
```
- id: 主键
- email: 邮箱（唯一）
- phone: 手机号（唯一）
- password_hash: 密码哈希
- nickname: 昵称
- avatar: 头像URL
- gender: 性别
- birthday: 生日
- dietary_preferences: 饮食偏好（JSON）
- allergies: 过敏源（JSON）
- created_at: 创建时间
- updated_at: 更新时间
- last_login: 最后登录
```

#### 美食表 (foods)
```
- id: 主键
- name: 美食名称
- category: 分类（中餐/西餐/日料等）
- cuisine: 菜系（川菜/粤菜等）
- tags: 标签（JSON数组）
- image_url: 图片URL
- description: 描述
- ingredients: 食材（JSON）
- nutrition: 营养成分（JSON）
- cooking_time: 烹饪时间
- difficulty: 难度
- price_range: 价格区间
- is_spicy: 是否辣
- is_vegetarian: 是否素食
- is_healthy: 是否健康
- suitable_scenarios: 适用场景（JSON）
- rating: 平均评分
- rating_count: 评分人数
- created_at: 创建时间
```

#### 用户收藏表 (favorites)
```
- id: 主键
- user_id: 用户ID（外键）
- food_id: 美食ID（外键）
- created_at: 收藏时间
```

#### 推荐历史表 (recommendations)
```
- id: 主键
- user_id: 用户ID（外键）
- food_id: 美食ID（外键）
- context: 推荐场景（JSON：时段、天气、心情等）
- rating: 用户评分
- feedback: 用户反馈
- created_at: 推荐时间
```

#### 用户成就表 (achievements)
```
- id: 主键
- user_id: 用户ID（外键）
- achievement_type: 成就类型
- achievement_name: 成就名称
- description: 描述
- icon: 图标
- unlocked_at: 解锁时间
```

#### 游戏记录表 (game_records)
```
- id: 主键
- user_id: 用户ID（外键）
- game_type: 游戏类型（转盘/骰子/问答）
- result: 结果
- options: 选项（JSON）
- created_at: 游戏时间
```

### 5.2 数据库选型
- **主数据库**：PostgreSQL 14+
  - 关系型数据，支持复杂查询
  - JSON 字段支持灵活存储
  - 良好的性能和稳定性

- **缓存数据库**：Redis
  - 会话存储
  - 热点数据缓存
  - 限流计数

- **文件存储**：阿里云 OSS / AWS S3
  - 用户头像
  - 美食图片

### 5.3 数据量预估
- **初期**：用户 1万，美食 500，日推荐 5万
- **增长期**：用户 10万，美食 2000，日推荐 100万
- **数据库设计**：支持百万级用户，千万级推荐记录

## 6. API 需求概览

### 6.1 认证相关
- POST /api/auth/register - 注册
- POST /api/auth/login - 登录
- POST /api/auth/logout - 登出
- POST /api/auth/refresh - 刷新 Token
- POST /api/auth/forgot-password - 找回密码

### 6.2 用户相关
- GET /api/users/profile - 获取个人资料
- PUT /api/users/profile - 更新个人资料
- PUT /api/users/preferences - 更新偏好设置
- GET /api/users/stats - 获取用户统计

### 6.3 美食相关
- GET /api/foods - 获取美食列表
- GET /api/foods/:id - 获取美食详情
- GET /api/foods/search - 搜索美食
- GET /api/foods/categories - 获取分类
- GET /api/foods/recommend - 智能推荐

### 6.4 收藏相关
- GET /api/favorites - 获取收藏列表
- POST /api/favorites - 添加收藏
- DELETE /api/favorites/:id - 取消收藏

### 6.5 推荐相关
- GET /api/recommendations/history - 推荐历史
- POST /api/recommendations - 记录推荐
- POST /api/recommendations/:id/rate - 评分

### 6.6 游戏相关
- POST /api/games/spin - 转盘游戏
- POST /api/games/dice - 骰子游戏
- POST /api/games/quiz - 问答游戏
- GET /api/games/history - 游戏历史

### 7.1 后端技术栈
- **框架**：Node.js + Express / NestJS
- **数据库**：PostgreSQL + Prisma ORM
- **缓存**：Redis
- **认证**：JWT + Passport
- **文档**：Swagger/OpenAPI
- **测试**：Jest
